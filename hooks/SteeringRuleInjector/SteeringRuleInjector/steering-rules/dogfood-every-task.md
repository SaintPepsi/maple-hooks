Dogfood Every Task. Each task in a plan should produce dogfoodable results — something that can be used, tested, or validated in a real session immediately after implementation. If a task only produces code that sits untested until a later integration step, restructure it so it's independently verifiable. "Does this work when I use it?" is the bar, not "does this compile?"
